UI Component Layer (for your B&W aesthetic)

Tone.js has no UI — that's the point. You'll need these pieces:

Knobs / sliders react-knob-headless (headless, fully styleable), or build SVG knobs — rotate on drag, no library required, total aesthetic control.
Envelope editor Custom SVG path component — 4 draggable handles (A, D, S, R) updating Tone.js envelope params in real time.
Oscilloscope / spectrum Tone.js ships Analyser and FFT nodes — render via Canvas or SVG. Raw waveform display works perfectly for minimal aesthetics.
FM routing matrix SVG grid or node graph (React Flow / xyflow) showing operator-to-operator modulation routing — very Operator-like, fully ownable aesthetically.
Suggested Build Path

Install Tone.js, wire to WebMIDI

npm install tone — Create a PolySynth(FMSynth) instance. Connect WebMIDI noteOn/noteOff to triggerAttack / triggerRelease. You have a playable FM instrument in under 30 lines.

Build a patch state object

Design a JSON schema for a synth patch: oscillator type, harmonicity, modulation index, ADSR values, filter cutoff, effects wet/dry. This becomes your serialization format — save/load presets per MIDI track.

Build the sound designer UI in React

Start with an ADSR envelope SVG editor and oscillator selector. Add FM parameter controls (harmonicity, modulation index). Keep it monochrome — coarse grid lines, heavy mono type, waveform preview as the hero visual element.

Add Tuna.js effects rack

Layer Tuna.js on top for reverb, delay, and distortion — route Tone.js output through Tuna nodes before reaching the destination. Design an effects chain UI that matches your panel aesthetic.

Later: Faust for custom DSP

Once the core UX is solid, explore FaustWasm for instruments that can't be assembled from Web Audio primitives — physical modeling, waveshaping algorithms, spectral synthesis. Faust-compiled nodes slot into the same AudioContext graph.
